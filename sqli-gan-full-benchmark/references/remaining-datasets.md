## Why SQLiV3 Needs the Normalized File

Raw `SQLiV3.csv` has header shape `Sentence,Label,,`, which pandas reads as:

- `Sentence`
- `Label`
- `Unnamed: 2`
- `Unnamed: 3`

Some rows shift the true label into `Unnamed: 2` or `Unnamed: 3` because SQL
payload fragments contain unquoted commas. The normalized file was generated by:

1. Finding the rightmost valid `0/1` label among `Label`, `Unnamed: 2`, and
   `Unnamed: 3`.
2. Joining all preceding non-empty fragments back into `Sentence`.
3. Dropping only rows with empty normalized text.
4. Writing a clean two-column CSV: `Sentence,Label`.

Observed normalized label distribution:

- `0`: 19,537
- `1`: 11,381
- Total: 30,918
